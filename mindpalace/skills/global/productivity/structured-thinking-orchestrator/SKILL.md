# Structured Thinking Orchestrator

Describe a situation and goal. This designs a tailored reasoning workflow — an ordered sequence of 2–5 thinking tools — then executes each step, feeding what each reveals into the next. The value is compounding clarity, not a single tool's take.

## When to use

Any time a problem is too tangled for one lens: a hard decision, a stuck problem, a human/political situation, a plan to evaluate, a strategy to think through, or something to communicate. If the user just needs one tool, route to a single reasoning skill instead.

## Process

1. **Intake.** Establish three things: the goal (what success looks like), the obstacle (what makes it hard), the decision point (what they're about to do/decide/communicate). If unclear, ask one targeted follow-up. State your one-sentence framing and confirm it before designing the plan.

2. **Design the workflow.** Order 2–5 distinct steps where each output informs the next. Principles:
   - Frame before analyzing (surface assumptions / observe first).
   - Generate before evaluating (option-mapping, alternatives before criteria-weighting, logic-check).
   - Understand people before designing for them (motivation/audience modeling before objection mapping).
   - Validate before committing (premortem, constraint-hardness, logic-fixer).
   - End with synthesis for understanding goals (systems leverage, frame analysis).

3. **Present the plan.** Show the ordered steps with rationale and the connection between each, then ask: run it, adjust it, or jump to one step.

4. **Execute with chaining.** Run step one. After each step output a brief handoff ("Key finding: …"), then offer 2–3 logical next steps plus a wrap-up option. Run each step on the situation plus all accumulated output — never re-explain from scratch.

5. **Synthesize.** Close with: what the full sequence revealed (3–5 insights no single step showed), the central tension/trade-off, one concrete recommended next action, and any load-bearing-but-uncertain assumption to revisit.

## Common workflow patterns

| Goal type | Typical sequence |
|---|---|
| Make a complex decision | surface-assumptions → option-mapping → criteria-weighting → premortem |
| Solve a stuck problem | surface-assumptions → lateral-thinking → alternatives → constraint-hardness |
| Understand a human situation | motivation-mapping → power-mapping → audience-modeling → empathy-circle |
| Communicate something hard | audience-modeling → objection-mapping → clarity-audit |
| Evaluate a plan/proposal | logic-check → premortem → impact-scan → constraint-hardness |
| Think through a strategy | systems-leverage → terrain → positioning → premortem |
| Write or reshape something | issue-audit → restructure → tone-alignment → line-editing |
| Understand a system/pattern | feedback-mapping → leverage-analysis → historical-precedent |

## Notes

- The goal drives the plan, not the problem type — "decide X" warrants different workflows depending on whether the goal is speed, thoroughness, buy-in, or ethics.
- 2–3 tightly chained steps usually beat a sprawling 5-step plan with thin connections.
- Adapt as you go: if step 2 reveals a different situation than step 1 assumed, redesign the remaining steps.
- The handoff between steps IS the skill — never skip the explicit connection from one step to the next.
