# Systems Thinking Toolkit

Applies systems thinking to complex, dynamic situations. Confirm the system and the behavior/dynamic in focus in one sentence, then route.

## Which tool fits

| You need to... | Tool |
|---|---|
| Recognize what recurring pattern this is | Archetype Matching |
| Understand unexpected/counterintuitive behavior | Emergence Detection |
| Map the feedback loops driving behavior | Feedback Mapping |
| Find where small interventions yield large change | Leverage Analysis |

When unclear → Feedback Mapping; the loops reveal archetypes, emergence, and leverage points together.

## Archetype Matching

Systems recur in patterns. Common archetypes: Limits to Growth (growth stalls at an unseen constraint), Shifting the Burden (quick fixes treat symptoms while the real problem grows), Fixes that Fail (interventions create delayed side effects), Escalation (each side responds, spiraling), Tragedy of the Commons (individually rational behavior depletes a shared resource). Name candidates; test each against the situation.
**Output:** Archetype with structural evidence, what it predicts next, and its specific leverage point.

## Emergence Detection

Emergent behavior arises from interactions, not components. When the system surprises everyone: map the interactions generating it. Which loop amplifies it? Which constraint is it bumping against? Pushing harder in the obvious direction often makes it worse — find the non-obvious intervention.
**Output:** The behavior described structurally, the interactions generating it, why intuitive fixes fail, and the non-obvious leverage point.

## Feedback Mapping

Identify reinforcing loops (R: A↑→B↑→A↑, exponential growth/collapse) and balancing loops (B: A↑→B responds to reduce A, stability/oscillation). Which loops are active? Dominant? What time delays exist — delays make oscillation likely. System behavior IS the behavior of its loops.
**Output:** Feedback map — reinforcing and balancing loops, dominant loops, time delays, and the behavior each generates.

## Leverage Analysis

Apply Donella Meadows' hierarchy. Low leverage: numbers, parameters, flow rates. Medium: loop strength, information availability, rules. High: system goals, structure, paradigm. Map current interventions by level. Find the high-leverage points being avoided — usually avoided for a reason (resistance, long horizon, belief change). That reason is worth examining.
**Output:** Intervention map by leverage level, high-leverage points being avoided and why, and the most feasible high-leverage move.
