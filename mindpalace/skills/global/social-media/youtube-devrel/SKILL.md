# YouTube for Developer Relations

YouTube is the second-largest search engine and where developers learn. It's a long game — consistency beats perfection.

## Content types

Tutorials (build X) — very high view potential, evergreen search traffic. Problem/solution and tool comparisons — high, search-driven. Live coding — engagement-focused. Project updates — community. Conference talks — repurpose. Shorts — discovery reach. Match length to audience: quick solutions 5-15 min, complete tutorials 15-45 min, deep dives 30-60 min, demos 3-10 min.

## Tutorial structure

`0:00 hook (what you'll build) → 0:30 prerequisites/setup → 2:00 concept (why) → steps → final demo → recap & next steps → CTA`. Hook in first 30s: show the end result, state what they'll learn, mention prerequisites, give a time estimate.

Pacing: short videos 1-2 min segments, standard 3-5 min, deep dives 5-10 min. Avoid: too much setup (skip ahead), not showing the end result, going too fast (pause on key steps) or too slow (edit out pauses), no timestamps (add chapters), walls of code (build incrementally).

## Recording setup

**Audio matters more than video** — bad audio = instant click-away. Priority: clear audio > readable code > lighting > 1080p+ resolution. Settings: 1920x1080 or 1440p, editor font 18-24pt, dark theme, zoom for key sections, disable all notifications, clean browser profile. Workflow: script key points (don't read verbatim), test audio levels, record in segments, leave pauses for edits, show face at transitions, capture b-roll. Budget: Blue Snowball + OBS + DaVinci Resolve (all free-ish); better: SM7B + ScreenFlow + Premiere.

## Live coding

Formats: Twitch→YouTube (1-4h), YouTube Live (30-90 min), pre-recorded "live" (20-45 min). Do: explain your thinking, embrace mistakes, read chat, have a goal, take breaks, add post-stream timestamps. Don't: code silently, pretend omniscience, ignore audience, meander, leave as one giant blob.

## Shorts vs long-form

Shorts (<60s, 30-45s optimal): immediate hook (1-2s), 9:16 vertical, one concept, big on-screen captions, clear voiceover. Ideas: quick tips, syntax shortcuts, tool demos, before/after refactors, hot takes. Strategy: 3-5/week for subscriber growth, tease long-form, repurpose cuts from long videos. Long-form = search traffic, watch-time revenue, depth, loyal subs; shorts = discovery, breadth, scroll-by subs.

## Thumbnails

3-5 words max, expressive face, tech logo/icon, high contrast brand colors, rule-of-thirds. Patterns: face + logo + emotion; before/after; big text + icon ("STOP" + X); comparison; result preview. A/B test: upload with thumbnail A, switch to B after 48h, compare CTR, keep the winner. Tools: Canva, Figma, Photoshop.

## YouTube SEO

Keyword locations by impact: title (highest, keyword early), description (first 2 sentences keyword-rich), captions/transcript (medium, edit auto-generated), tags (low). Title formulas: "[Tech] Tutorial: Build [X]", "How to [Task] with [Tech]", "[A] vs [B]: Which Should You Learn?", "[N] [Topic] Tips in [Time]", "Why I [Action] [Tech]". Description: 150-char keyword summary, timestamps, resources, links, hashtags. Research via YouTube autocomplete, TubeBuddy/VidIQ, Google Trends, competitor titles.

## Do / Don't

Do: show end result first, add chapters, readable code fonts, invest in audio, consistent schedule, engage comments, repurpose to shorts, custom thumbnails. Don't: skip the hook, tiny fonts, room echo, forget CTAs, post inconsistently, ignore SEO, clickbait thumbnails (trust erosion), or chase perfection (ship it).

## First-video checklist

Outline key points; test audio + screen recording; record 1080p+; 18pt+ editor font; end result in first 30s; timestamps/chapters; custom thumbnail; SEO title/description; CTA at end; reply to comments within 24h. Tools: OBS, DaVinci Resolve, Descript, Canva, TubeBuddy, Riverside.fm.
