# X/Twitter for Developers

X is the real-time pulse of the dev community. It rewards authenticity and technical depth.

## What works & the algorithm

High engagement: technical threads (saved/shared), code screenshots, hot takes (risky), building in public, memes (know your audience). Low: plain links (deprioritized), bare retweets.

Algorithm signals (high impact): replies in first hour, time spent on post (threads > tweets), profile visits, bookmarks. Negative: external links. So front-load value on-platform; put links late.

## Thread structure

Anatomy: Tweet 1 hook → tweet 2 context/promise → tweets 3-8 the meat (examples/steps/insights) → summary/takeaway → CTA + engagement ask.

Hook patterns: contrarian ("Hot take: you don't need Kubernetes for most apps"), promise ("How we cut our AWS bill 60%"), story ("Our API went down 4 hours. Here's what happened:"), list tease ("7 TypeScript tricks:"), question, result.

Best practices: 5-12 tweets; first tweet is hook only (no links/hashtags); one idea per tweet; number with X/10 or emoji bullets; code as screenshots; mix short and explanatory; end with CTA (follow/reply/bookmark).

## Code screenshots

Screenshots beat code blocks: syntax highlighting, controlled appearance, context, more visual, higher engagement. Tools: Carbon, Ray.so, Snappify, CodeSnap (VS Code), Silicon (CLI). Best practices: dark theme, include filename, highlight key lines, readable width, consistent styling, comments in the code.

## Timing & frequency

Post (PT): 6-8am (US East + Europe), 10am-12pm peak, 4-6pm evening. Frequency: 1-3 tweets/day sustainable, 3-5 growth mode; 1 thread/week good cadence, 2-3 aggressive. Post threads in morning, engage replies for first 2h, quote-tweet your thread later, post follow-up same day.

## Hashtags (sparingly)

Max 1-2 per tweet, at the end, skip in threads. Relevant dev tags: #buildinpublic, #devrel, #100DaysOfCode, #opensource, plus language tags.

## Growth flywheel & profile

Flywheel: valuable content → engagement → engage replies → reply to others (get discovered) → consistency (algorithm favors) → followers grow → repeat. Profile: clear face photo, real/consistent name, bio = `[Role] at [Company]. Building [what]. Tweeting about [topics]. [Credibility]`, one important link, pinned best content.

Reply strategy: reply to big accounts in your niche (visibility), answerable questions (expertise), your followers (relationships), and controversial takes carefully. Add info, share experience, ask thoughtful follow-ups, disagree with reasoning.

## Avoid the cringe zone

Fails: humble brags, engagement bait ("like if you agree"), empty motivation, obvious pitches, excessive hashtags, copied threads, AI slop, overused "thread guy" formulas. Avoid buying followers, engagement pods, follow/unfollow churn, DM pitching, public arguing, constant dunking, deleting unpopular takes. Authenticity test before posting: would I say this in person, is it genuinely useful, am I adding to the conversation, would I respect someone who posted this.

## Content mix

~50% educational/useful, 25% personal/journey, 15% opinions/takes, 10% promotional.

## Thread checklist

Hook compelling (no links/hashtags); each tweet a complete thought; code in screenshots; 5-12 tweets; TL;DR included; CTA at end; scheduled for optimal time; ready to engage first 2h. Tools: Carbon, Typefully (drafting/scheduling), Buffer, TweetDeck.
