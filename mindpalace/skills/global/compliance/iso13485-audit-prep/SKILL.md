# ISO 13485 QMS Audit Prep — Forcing Questions

Six traceability-obsessed questions before any internal audit, MDR/FDA QSR review, or product launch. Run also after significant CAPA closure, post-recall, or quarterly during submission prep.

## The six questions

1. **Pull three random DHFs — is design verification + validation evidence complete?** (Most-cited finding.) DHF includes design plan + inputs + outputs + verification + validation + transfer + changes. Sample stratified by product class (I, IIa, IIb, III per MDR). Verify traceability matrix from user needs through clinical evidence.
2. **Last 5 CAPAs with effectiveness verification.** (Second-most-cited.) Containment/correction/corrective-action distinction, root cause 5-Why min, effectiveness = measurable evidence, closure approved, repeat CAPAs across products = systemic trigger.
3. **When was process validation (IQ/OQ/PQ) last revalidated?** (Clause 7.5.6 — often stale.) Revalidate on process/equipment/material change or periodic schedule; SPC trend monitoring where statistics apply (Clause 8.4). Cross-check 21 CFR 820.75.
4. **Risk management file for the highest-risk product.** (Clause 7.1 + ISO 14971:2019.) RM plan per product, hazard ID covers foreseeable misuse, control hierarchy applied (inherent safety > protective measures > information), residual risk evaluated + accepted with rationale, post-production info feeds back. AI-enabled devices: layer ISO 42001 A.5 impact assessment.
5. **Post-market surveillance evidence — last 6 months.** (Clause 8.2.1.) Complaint log + investigation closure, vigilance (serious incident/FSCA) per regulation, trend analysis + management-review input, PMCF for MDR high-risk, MDR reports per 21 CFR 803 for US-marketed.
6. **Management-review evidence covering all Clause 5.6 inputs.** 5.6.2 inputs: audit results, customer feedback, process performance, product conformity, preventive/corrective-action status, prior-review follow-up, QMS-affecting changes, improvement recommendations, regulatory requirements. Outputs 5.6.3: improvement decisions, product-requirement changes, resource needs.

## Output

Decision (programme-plan / DHF-closure / CAPA-health / post-market-trend / pre-cert / MDR-FDA-alignment) → design-control status per sampled DHF (verification/validation/clinical/traceability) → CAPA health (root cause depth, effectiveness, aging >90d, repeat issues) → process validation (on schedule %, stale, statistics per 8.4) → risk file (sampled RMFs, post-production updates, residual acceptance signed) → post-market (complaint trending, vigilance timeliness, PMCF) → management review (last date, 5.6.2 inputs, overdue actions) → cross-framework (EU MDR, FDA QSR post-Feb 2026 harmonization, ISO 42001 if AI-enabled) → verdict 🟢 READY / 🟡 CLOSE-DHF-GAPS / 🔴 NOT-READY → top 3 actions.
