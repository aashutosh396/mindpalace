# Company Operating System

Most operational dysfunction is a system problem, not a people problem. Every company has an OS — making it explicit makes it improvable.

## The six core components (any framework has these)

### 1. Accountability chart
Not an org chart — answers "who owns this outcome?" One person owns each function (multiple may work in it). No shared ownership ("Alice and Bob both own it" = nobody owns it). One person can own multiple seats early. Revisit quarterly. Build in a workshop: list functions → assign one owner each → find gaps/overlaps → publish.

### 2. Scorecard
Weekly metrics (not monthly/quarterly). 5–15 max. Each has an owner and a hard weekly target (a number, not a range). Red/yellow/green, no paragraphs. Only red metrics get discussion time. Anti-pattern: tracking 40 KPIs = watching, not managing.

### 3. Meeting pulse
Daily standup (15min, blockers only) · Weekly L10/leadership sync (90min, scorecard + issues) · Monthly dept review (OKR progress) · Quarterly planning (1–2 days, rocks) · Annual planning (2–3 days, 1yr + 3yr vision).
**L10 fixed agenda:** good news (5) → scorecard, flag red (5) → rock review (5) → customer/employee headlines (5) → issues list IDS (60) → to-dos (5) → rate the meeting 1–10 (5).

### 4. Issue resolution (IDS)
Max 15 min/issue. **Identify** the root cause (one sentence) → **Discuss** (time-boxed; stop when repeating) → **Solve** (one owner, one action, one due date). Anti-patterns: "take it offline" (never resolved), discussing without deciding, revisiting decided issues. Maintain a prioritized Issues List, pruned weekly.

### 5. Rocks (90-day priorities)
3–7 per person max (more = none done). Binary: done or not (no "60% complete"). Set quarterly, reviewed weekly. Bad: "improve our sales process." Good: "Implement Salesforce with full pipeline stages + weekly reporting by March 31." A to-do is one action; a rock is 90 days of work.

### 6. Communication cadence
All-employee monthly update + quarterly all-hands · weekly leadership scorecard · monthly board memo · investor update · per-release customer notes. Default: when unsure, share — under-communication always costs more than over-communication internally.

## OS selection

10–250, founder-led chaos → EOS/Traction · ambitious growth needing strategy cascade → Scaling Up · tech/hypothesis-driven → OKR-native · flat/high-autonomy → Holacracy (if patient) · else → custom hybrid.

## Quick start (first 30 days)

1. Build the accountability chart (2h workshop). 2. Define 5–10 weekly scorecard metrics (1h). 3. Start the weekly L10 (no prep, just start).

## Failure modes

Partial implementation (half an OS is worse than none — theater without accountability) · meeting fatigue (replace meetings, don't add) · metric overload (start with 5) · rock inflation (hard limit 7) · leader non-compliance (the OS mirrors the respect leaders give it) · annual planning without quarterly review.
