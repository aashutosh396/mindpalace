# Copy Editing — The Seven Sweeps

Systematically improve existing copy through focused passes while preserving the core message. Editing enhances, it doesn't rewrite. Multiple focused passes beat one unfocused review; each edit has a clear reason; preserve the author's voice.

## The Seven Sweeps
Run sequentially. After each sweep, loop back to confirm prior sweeps aren't compromised.

1. **Clarity** — can the reader understand it? Kill confusing structures, unclear pronouns, jargon, ambiguity, missing context. Watch for sentences doing too much, abstract language, assumed knowledge, buried points. After: confirm "Rule of One" (one idea/section) and "You Rule" (speaks to reader).

2. **Voice & Tone** — is it consistent? Catch formal↔casual shifts, inconsistent personality, jarring mood changes, off-brand word choices. Read aloud to hear breaks. After: re-check Clarity.

3. **So What** — does every claim answer "why should I care?" For each statement ask "so what?" If no deeper benefit, add a "which means…" bridge. ❌ "AI-powered analytics" ✅ "AI analytics surface insights you'd miss — so you decide in half the time." After: re-check Voice, Clarity.

4. **Prove It** — is every claim supported? Flag unsubstantiated claims, missing social proof, "best/leading" without evidence. Proof types: named testimonials, case studies, stats with sources, third-party validation, guarantees, logos, review scores. After: re-check So What, Voice, Clarity.

5. **Specificity** — is it concrete? Replace vague ("improve," "many," "fast") with specifics. Save time → Save 4 hours/week. Many customers → 2,847 teams. Great support → Response within 2 hours. Remove what can't be made specific (it's filler). After: re-check Prove It → Clarity.

6. **Heightened Emotion** — does it make the reader feel? Add emotional texture to flat sections: pain of current state, frustration with alternatives, FOMO, desire for transformation, relief. Techniques: paint the "before" vividly, sensory language, micro-stories, reflective questions. Keep authentic, not manipulative. After: re-check Specificity → Clarity.

7. **Zero Risk** — every barrier to action removed? Near CTAs: address objections, add trust signals, clarify next steps, remove hidden costs. Risk reducers: money-back, free trial, "no credit card," "cancel anytime," social proof near CTA. After: full reverse pass through all sweeps.

## Quick-pass checks (when full sweeps aren't needed)
- **AI-pattern check** — if AI-drafted, humanize first; polishing AI mush yields polished AI mush.
- **Cut words**: very, really, extremely, just, actually, basically, "in order to," unnecessary "that," things/stuff.
- **Replace**: utilize→use, implement→set up, leverage→use, facilitate→help, innovative→new, robust→strong, seamless→smooth.
- **Sentence**: one idea each; vary length; front-load key info; max ~25 words.
- **Paragraph**: one topic; 2-4 sentences for web; strong openers; white space.

## Common problems → fixes
- Wall of features → add "which means…" bridges.
- Corporate speak → "how would a human say this?"
- Weak opening → lead with reader's problem/outcome.
- Buried CTA → make it obvious, early, repeated.
- No proof → add specific testimonials/numbers.
- Generic claims → specify who/how/how much.
- Mixed audiences → pick one, write to them.

## Working with sweeps
Run a sweep → present findings (what's wrong + why it hurts conversion + how to fix) → recommend specific edits → let author decide → re-check earlier sweeps → repeat until a full sweep finds nothing. Deliver sweep-by-sweep, prioritized by conversion impact — not all at once.
