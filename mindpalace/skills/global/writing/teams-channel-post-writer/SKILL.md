Create well-structured, educational Teams channel posts for internal knowledge sharing — feature announcements, tips, prompting patterns, lessons learned.

## Workflow
1. **Understand the topic.** Research thoroughly via official docs; verify release dates/version numbers from changelogs; identify the core benefit/principle; collect concrete examples from real usage. Checklist: found official date/version? verified behavior? identified authoritative links? understood the underlying principle?
2. **Plan the content:** Hook (what's new/important?), Core principle (what best practice it illustrates), Examples (concrete prompts/workflows), Call-to-action (what to try next).
3. **Draft** using the structure: emoji title + clear description → intro (release date + context) → "What it is" (1-2 sentences) → "How to use it" (Normal vs Better pattern + explicit instructions) → "Why use it" (underlying principle + ~4 benefits) → 3+ realistic concrete examples → options/settings → call-to-action → "Learn more" links.
4. **Apply guidelines:** educational/helpful tone; "Normal/Better" (not "Wrong/Correct"); concrete realistic examples; explain the "why"; clear structure with bullets; verified facts/dates.
5. **Save** with a descriptive filename.

## Key principles
- **Show, don't just tell.** Always include copy-pasteable examples. Use "Normal vs Better" comparisons so readers don't feel criticized.
- **Connect to principles.** Don't just describe features — explain the best practice behind them (e.g. tie an Explore agent to "context offloading").
- **Make it actionable.** Be explicit about invocation patterns; users should copy/paste and immediately use.
- **Verify everything.** Research release dates, verify behavior, link authoritative sources — accuracy builds trust.
