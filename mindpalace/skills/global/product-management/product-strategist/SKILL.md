# Product Strategist

Strategic toolkit for Head of Product: vision, alignment, OKR cascade from company → product → team.

## Quarterly Strategic Planning

### 1. Define strategic focus

| Strategy | When to use |
|---|---|
| Growth | scaling users, market expansion |
| Retention | reducing churn, improving LTV |
| Revenue | increasing ARPU, new monetization |
| Innovation | differentiation, new capabilities |
| Operational | efficiency, scaling operations |

### 2. Gather input metrics

Current/target for the headline metric (e.g. MAU 100k → 150k), plus secondary (NPS, etc.).

### 3. Cascade OKRs

Company OKRs → Product OKRs (with explicit "Supports: CO-X" links + a product contribution % to each company OKR, default 30%) → Team OKRs (distributed across actual org teams). Limit 3-5 objectives per level, each with 3-5 measurable KRs (current + target values).

### 4. Review alignment scores

| Score | Target | Action if below |
|---|---|---|
| Vertical alignment | >90% | ensure every objective links to a parent |
| Horizontal alignment | >75% | check team-coordination gaps |
| Coverage | >80% | validate all company OKRs are addressed |
| Balance | >80% | redistribute if one team overloaded |
| **Overall** | **>80%** | <60% needs restructuring |

### 5. Refine, validate, export

- [ ] Review objectives with stakeholders
- [ ] Adjust team assignments to capacity
- [ ] Validate contribution % realistic
- [ ] No conflicting objectives across teams
- [ ] Set tracking cadence (bi-weekly check-ins)

## Best Practices

- **Cascade** — validate parent-child relationships before finalizing; KRs must be measurable.
- **Alignment** — target >80% overall; investigate <60%; balance prevents overload; horizontal alignment prevents conflicting goals.
- **Team config** — match actual org; adjust contribution % by team size; platform/infra teams support all objectives, specialized (ML/Data) only relevant ones.
