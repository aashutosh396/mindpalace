# Product Discovery

Run structured discovery to find high-value opportunities and de-risk product bets.

## Core Workflow

1. **Define desired outcome** — one measurable outcome to improve; set baseline + target horizon.
2. **Build Opportunity Solution Tree** — Outcome → opportunities → solution ideas → experiments. Keep opportunities grounded in user evidence, not internal opinion.
3. **Map assumptions** — categorize as desirability/viability/feasibility/usability; score by risk × certainty.
4. **Validate the problem** — interviews + behavior analysis; confirm frequency, severity, willingness to solve; reject weak opportunities early.
5. **Validate the solution** — prototype before building; concept/usability/value tests; measure behavior not stated preference.
6. **Plan discovery sprint** — 1-2 week cycle with explicit hypotheses, daily evidence reviews, end with decision: proceed / pivot / stop.

## Opportunity Solution Tree (Teresa Torres)

- Outcome = metric to move; Opportunities = unmet needs/pains; Solutions = candidate interventions; Experiments = fastest learning actions.
- Quality checks: ≥3 distinct opportunities before converging; ≥2 experiments per top opportunity; tie every branch to an evidence source.

## Assumption Mapping

Categories: Desirability (users want this), Viability (business value exists), Feasibility (team can build/operate), Usability (users can successfully use). **Prioritization rule:** high-risk + low-certainty assumptions tested first.

## Problem Validation

Techniques: problem interviews on current behavior, journey friction mapping, support-ticket + sales-call synthesis, behavioral-analytics triangulation. Evidence thresholds: same pain across multiple target users, observable workaround behavior, measurable cost of current pain.

## Solution Validation

Concept tests (value-prop comprehension), prototype usability tests (task success / time-to-complete), fake-door/concierge (demand signal), limited beta cohorts (retention/activation signals).

## Discovery Sprint (10-day)

Day 1-2 outcome + opportunity framing · Day 3-4 assumption mapping + test design · Day 5-7 problem + solution tests · Day 8-9 evidence synthesis + decision options · Day 10 stakeholder decision review.
