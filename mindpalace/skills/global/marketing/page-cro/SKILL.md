# Page Conversion Rate Optimization

Analyze marketing pages and provide actionable recommendations to improve conversion.

## Initial assessment
Page type (homepage/landing/pricing/feature/blog/about) · primary conversion goal (signup/demo/purchase/subscribe/download/contact) · traffic context (organic/paid/email/social).

## Analysis framework (in order of impact)
1. **Value proposition clarity** (highest) — understandable in 5 seconds? primary benefit clear, specific, differentiated? in customer language? Common issues: feature-focused, too vague/clever, trying to say everything.
2. **Headline effectiveness** — communicates the value prop? specific? matches traffic source? Strong patterns: outcome-focused, specificity (numbers/timeframes), social proof.
3. **CTA placement, copy, hierarchy** — one clear primary action, visible without scrolling, value-communicating copy (not "Submit"); logical primary/secondary; repeated at decision points.
4. **Visual hierarchy & scannability** — main message clear when scanning? important elements prominent? white space? images support not distract?
5. **Trust signals & social proof** — logos, attributed testimonials with photos, case-study numbers, review scores, security badges; place near CTAs and after claims.
6. **Objection handling** — price/value, "will it work for me?", implementation difficulty, "what if it fails?"; address via FAQ, guarantees, comparisons, process transparency.
7. **Friction points** — too many fields, unclear next steps, confusing nav, unnecessary required info, mobile issues, load times.

## Output format
Lead with a conversion-audit score, then:
- **Quick wins** (implement now) — easy, likely-immediate impact.
- **High-impact changes** (prioritize) — bigger effort, significant improvement, each with rationale + effort + success metric.
- **Test ideas** — hypotheses worth A/B testing rather than assuming.
- **Copy alternatives** — 2-3 per key element (headline, CTA) with rationale.

Never recommend A/B testing as a substitute for obvious fixes — call out what to fix vs what to test.

## Page-specific frameworks
Homepage (positioning for cold visitors, quick path to common conversion, serve ready + researching). Landing (message match, single CTA, complete argument on one page). Pricing (clear comparison, recommended plan, "which is right for me?" anxiety). Feature (feature→benefit, use cases, try/buy path). Blog (contextual inline CTAs at natural stops).

## Proactive triggers
"This page isn't converting" / low conversion / high bounce → activate framework. New landing page being built → offer pre-launch CRO review. Paid traffic mentioned → flag message-match + single-CTA. Pricing discussion → recommend pricing-page CRO. A/B results reviewed → generate next-round hypotheses.
