Create popups that convert without annoying users or damaging brand. (For forms in popups see form-cro; for whole-page conversion see page-cro.)

## Assess first
Purpose (email capture / lead magnet / discount / announcement / exit save / feature promo / survey), current performance + triggers + complaints, traffic context (source, new vs returning, page types).

## Per-popup spec (always specify all)
Type · Trigger (when it appears) · Targeting (who sees it) · Frequency (how often) · Copy (headline, subhead, CTA, decline text — complete set) · Design notes (layout, imagery, mobile). If multiple popups: define conflict rules so they don't overlap.

## Strategies by business
- **E-commerce:** entry/scroll first-purchase discount → exit-intent bigger discount → cart-abandonment recovery.
- **B2B SaaS:** click-triggered demo/lead magnet → scroll newsletter → exit-intent trial reminder.
- **Content/media:** scroll-based newsletter → page-count subscribe → exit-intent "don't miss future content."
- **Lead gen:** time-delayed list-building → click-triggered lead magnets → exit-intent final capture.

## Experiment levers
- **Placement/format:** top bar vs banner, sticky vs static, center modal vs corner slide-in, full-screen vs small.
- **Triggers:** exit-intent vs time delay (10/30/60s) vs scroll depth (25/50/75%) vs page count vs behavior/source.
- **Messaging:** urgency vs value copy, CTA text/color, single vs primary+secondary CTA, decline tone, countdown timers, social proof.
- **Personalization:** new vs returning, segment by source, progressive profiling, exclude already-converted.
- **Frequency rules:** capping (per session vs per week), cool-down after dismissal, escalating offers across visits.

## Compliance (flag proactively)
GDPR consent + opt-in mechanics + privacy link; Google intrusive-interstitials policy (especially mobile).
