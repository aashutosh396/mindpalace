Convert free→paid or upgrade tiers at moments when the user has experienced enough value to justify commitment. (For the public pricing page use page-cro.)

## Assess first
Upgrade context (freemium→paid / trial→paid / tier upgrade / usage limit), product model (what's free vs gated, current conversion rate), user journey (when it appears, what they've experienced).

## Core principles
1. **Value before ask** — show the paywall after the aha moment, never before.
2. **Show, don't just tell** — preview/demonstrate what they're missing.
3. **Friction-free path** — easy to upgrade when ready.
4. **Respect the no** — easy to continue free; trust preserves future conversion.

## Trigger points
- **Feature gate** — user clicks paid-only feature: explain why it's paid, show what it does, quick unlock, option to continue without.
- **Usage limit** — clear "limit reached," show what upgrading provides, don't block abruptly.
- **Trial expiration** — early warnings (7/3/1 day), what-happens-on-expiry, summary of value received.
- **Time-based** — after X days: gentle reminder, highlight unused paid features, easy dismiss.

## Screen components (deliver all as a complete set)
Headline ("Unlock [Feature] to [Benefit]") · Value demonstration (preview / before-after) · Feature comparison (current plan marked) · Pricing (clear, annual vs monthly) · Social proof · CTA ("Start Getting [Benefit]") · Escape hatch ("Continue with Free").

## Timing & frequency
Show after value moment / activation / genuine limits. NEVER during onboarding, mid-flow, or repeatedly after dismissal. Limit per session; cool-down in days not hours; track annoyance signals.

## Anti-patterns to reject
Hidden close button · confusing plan selection · guilt-trip copy · asking before value delivered · too-frequent prompts · blocking critical flows.

## A/B test
Trigger timing, headline/copy, price presentation, trial length, feature emphasis, layout. Track: impression rate, CTR-to-upgrade, completion rate, revenue per user, post-upgrade churn.

## Mobile
Flag App Store IAP rules and Google Play billing requirements when relevant.
