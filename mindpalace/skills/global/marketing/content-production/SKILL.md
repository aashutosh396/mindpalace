# Content Production Pipeline

Take a topic from blank page to finished, optimized piece that ranks, converts, and gets read. This is the execution engine, not the strategy layer.

## Gather (ask in one shot)
Topic/working title · target keyword · audience (and what they already know) · goal (inform/convert/authority/trial) · approx length · existing content to link to. If the topic is vague ("write about AI"), push back for the specific angle.

## Three modes
**Mode 1 — Research & brief**, **Mode 2 — Draft**, **Mode 3 — Optimize & polish**. Run in sequence or jump to any.

### Mode 1: Research & brief
1. **Competitive analysis** — top 5-10 ranking pieces, map angles, find the gap, check search intent:
   - "What is / How to" dominate → informational → comprehensive guide.
   - Product pages/reviews → commercial → comparison/buyer's guide.
   - Forum results (Reddit/Quora) → discovery → opinionated piece with real POV.
2. **Source gathering** — 3-5 credible citable sources. If you can't cite a specific number, don't make a vague claim. "Studies show" is a red flag — find the study.
3. **Brief** — target + secondary keywords, reader profile + JTBD, angle/POV, required H2 structure, key claims to prove, internal links, pieces to beat.

### Mode 2: Draft
- **Outline first**: hook-worthy H1 (keyword + curiosity), 4-7 logical H2s, sparing H3s, CTA-adjacent conclusion.
- **Intro** (3-4 sentences): name the reader's situation → what this does about it → optional reason to trust you. Avoid "In today's digital landscape…" and burying the point.
- **Each H2**: state main point in sentence 1 → prove with example/stat/comparison → one actionable takeaway. Readers skim; every section delivers value alone.
- **Conclusion**: core-argument summary (1-2 sentences) → single most important next thing → CTA if relevant. Don't pad.

### Mode 3: Optimize & polish (run in order)
- **SEO**: title tag (keyword, <60 chars, curiosity), H1 ≠ title, 2-3 H2s with secondary keywords, keyword in first 100 words, descriptive alt text, short keyword-first slug.
- **Readability**: target 70+; avg sentence 15-20 words, no paragraph >4 sentences, explain jargon, active voice.
- **Brand voice**: check tone markers, sentence rhythm, vocabulary fingerprint against the brand profile; rewrite drift.
- **Structure audit**: intro delivers on headline? every H2 earns its place? ≥2 concrete examples? conclusion earned?
- **Internal links**: 2-4 min, descriptive anchor text (not "click here").
- **Meta tags**: meta description 150-160 chars + keyword + hook; OG title/description; canonical.

### Quality gates (don't publish until clean)
- [ ] Primary keyword appears naturally 3-5x (not stuffed).
- [ ] Every factual claim sourced or labeled opinion.
- [ ] ≥1 image/table/visual breaks up text.
- [ ] Intro doesn't open with a cliché.
- [ ] All internal links work.
- [ ] Readability ≥70.
- [ ] Word count within 10% of target.

## Proactive flags
Thin-content risk (600 words vs 2,000-word competitors) → surface before drafting. Keyword cannibalization → flag (splits authority). Intent mismatch → flag (traffic won't convert). Missing sources ("many companies") → flag each. CTA/goal disconnect → flag.
