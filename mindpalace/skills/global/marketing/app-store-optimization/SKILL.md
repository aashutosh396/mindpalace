App Store Optimization for iOS + Android: keyword research, metadata, competitor analysis, launch, A/B testing.

## Keyword research
1. Define audience + core app functions (problem solved, demographics, category).
2. Seed keywords from features, user language (not dev jargon), store autocomplete.
3. Expand with modifiers (free/best/simple), actions (create/track), audiences (for students/teams).
4. Evaluate each: search volume, competition, relevance.
5. Score & place: primary → Title + iOS keyword field; secondary → subtitle/short desc; tertiary → full description.

**Scoring weights:** Relevance 35% · Volume 25% · Competition 25% · Conversion intent 15%.
**Placement weight (high→low):** App Title → iOS Subtitle / iOS Keyword Field / Android Short Desc → Full Description.

## Metadata optimization
- Title formula: `[Brand] - [Primary Keyword] [Secondary Keyword]`.
- iOS keyword field: no spaces after commas, no plurals (Apple indexes both), no words already in title, no duplicates → maximize term count within 100 chars.
- Description structure: Hook (pain + value prop + primary kw) → Features (bullets + secondary kws) → Social proof → CTA.
- Target ~2-3% primary keyword density; never stuff (>5%).

**Character limits:** Title 30 (iOS)/50 (Android) · iOS Subtitle 30 · Android Short Desc 80 · iOS Keywords 100 · iOS Promo Text 170 · Full Desc 4000 · What's New 4000 (iOS)/500 (Android).

## Competitor analysis
Analyze 10+ competitors; extract keywords from titles/subtitles/first-100-words; build coverage matrix; find gaps (keywords with <40% competitor coverage, high-volume terms missed, long-tail). Audit visuals (icon, screenshots, video), ratings, review themes.

## A/B testing (Apple Product Page Optimization / Google Store Listing Experiments)
Prioritize by impact: Icon (10-25% lift) > Screenshot 1 (15-35%) > Title (5-15%) > Short Desc (5-10%). Test ONE variable, hit 95% significance, run ≥7 days. Sample sizes (per variant, by baseline CVR): 1%→31k, 2%→15.5k, 5%→6.2k, 10%→3.1k impressions.

## Launch (4-week runway)
T-4wk finalize metadata + assets + analytics + press kit. T-2wk submit, verify guideline compliance. Launch day: verify live in both stores, announce, respond to reviews within 24h. Days 1-7: track download velocity hourly. 7-day retro → first metadata update at T+2wk.

## Platform notes
iOS keyword changes require resubmission (promo text is editable without update). Android metadata indexes in 1-2h. Keyword volume = estimates only. **Don't use for:** web apps (use web SEO), enterprise/internal apps, TestFlight betas, or paid ad strategy.

## Proactive flags
No keyword in title (the #1 ranking factor) · screenshots show UI not value · no ratings strategy (sub-4.0 kills conversion) · keyword-stuffed description.
