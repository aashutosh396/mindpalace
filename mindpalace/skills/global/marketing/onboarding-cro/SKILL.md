Get users to their aha moment fast and build habits that drive retention. (For signup/registration flow use signup-flow-cro; for ongoing emails use email-sequence.)

## Assess first
1. Product context (B2B/B2C, core value prop). 2. Activation definition (the aha moment / earliest action correlated with retention). 3. Current state (what happens post-signup, where they drop off).

## Core principles
1. **Time-to-value is everything** — remove every step between signup and core value.
2. **One goal per session** — first session = one successful outcome; defer advanced features.
3. **Do, don't show** — interactive beats tutorial.
4. **Progress creates motivation** — show advancement, celebrate completions, make the path visible.

## Define activation
The action that correlates most with retention: what do retained users do that churned users don't? Examples — PM: create project + invite teammate · Analytics: install tracking + see first report · Design: create + export · Marketplace: first transaction. Track: % reaching activation, time-to-activation, steps-to-activation, activation by cohort/source.

## Flow design
**First 30 seconds — pick one:** Product-first (simple/B2C/mobile; risk = blank-slate overwhelm) · Guided setup (needs personalization; risk = friction before value) · Value-first (demo data; risk = doesn't feel real). Always: one clear next action, no dead ends, progress indication.

**Onboarding checklist** (use when multiple setup steps): 3-7 items ordered by value, start with quick wins, progress bar, celebration on completion, dismissable (don't trap).

**Empty states** = onboarding opportunities: explain the area, show what it looks like with data, clear primary action, optionally pre-populate example data.

**Tooltips/tours:** max 3-5 steps, dismissable, don't repeat for returning users.

## Multi-channel
Trigger emails reinforce in-app actions (don't duplicate): welcome (immediate), incomplete onboarding (24h, 72h), activation achieved (celebrate + next step), feature discovery (day 3/7/14). Email supports in-app; it never replaces it.

## Stalled users
Define stalled (X days inactive / incomplete setup) → re-engage via email sequence, in-app "pick up where you left off," human outreach for high-value accounts.

## Measure
Activation rate, time-to-activation, onboarding completion %, Day 1/7/30 retention. Run funnel analysis on step counts; the biggest-drop step is where the audit starts.

## Patterns by product type
B2B SaaS: setup wizard → first value → team invite → deep setup · Marketplace: profile → browse → first transaction → repeat · Mobile: permissions → quick win → push setup → habit loop · Content: follow/customize → consume → create → engage.

## Output: onboarding audit
Lead with activation score + named biggest-drop step. Per issue: Finding → Impact → Recommendation → Priority. Every recommendation names a specific step, metric, or trigger.
