# Sales Engineer — 5-Phase Workflow

## Phase 1 — Discovery & Research
Technical discovery calls; map current architecture and pain points; identify integration, security, and compliance requirements; assess competitive landscape. Score requirement coverage.
**Gate:** coverage >50% AND must-have gaps ≤3 before Phase 2.

## Phase 2 — Solution Design
Map capabilities to requirements; design integration architecture; identify customization effort; build differentiation strategy. Build a competitive matrix to surface differentiators and vulnerabilities.
**Gate:** at least one strong differentiator per customer priority; else escalate to product.

## Phase 3 — Demo Prep & Delivery
Build a demo environment matching the use case; script per stakeholder role; prepare objection handling and failure-recovery paths; capture feedback.
**Gate:** demo script covers every must-have requirement before delivery.

## Phase 4 — POC & Evaluation
Define scope, success criteria, timeline; allocate resources; phased testing (core → advanced → edge); track against criteria; generate a scorecard.
**Gate:** scorecard >60% across functionality, performance, integration, usability, support; else loop back to Phase 2.
Default phasing: Wk1 setup · Wk2-3 core testing · Wk4 advanced/edge · Wk5 evaluation + go/no-go.

## Phase 5 — Proposal & Closing
Compile POC results; build a technical proposal with implementation plan; address objections with evidence; support pricing/packaging; run win/loss analysis post-decision.

## Scoring conventions
- **Coverage:** Full 100% / Partial 50% / Planned 25% / Gap 0%. Priority weighting: Must-Have 3× / Should-Have 2× / Nice-to-Have 1×.
- **Bid/No-Bid:** Bid if coverage >70% AND must-have gaps ≤3; Conditional if coverage 50-70% OR gaps 2-3; No-Bid if coverage <50% OR gaps >3.
- **Feature scoring (competitive):** Full 3 / Partial 2 / Limited 1 / None 0.
